AI SEO 网页版：企业级产品改进方案 (v2.0 - Web Edition)0. 核心定位与转型目标平台化: 从单纯的“功能网页”转型为“企业级 AI SEO SaaS 平台”。全设备适配: 强化移动端与 PC 端的响应式体验（Mobile-First）。商业闭环: 建立完善的账号体系、额度消耗与多语言支撑。1. 视觉与交互重构 (UI/UX)1.1 SaaS White 极简风格配色方案:彻底移除所有深色背景（bg-dark-900 等）。主色调: 使用纯白 (#FFFFFF) 卡片背景。辅助色: 底色使用 #F8FAFC (Slate-50) 增加页面呼吸感。文字: 标题使用深色 #1E293B，正文使用 #64748B。响应式布局:针对网页版，需确保在不同屏幕尺寸下内容的自动重排。PC 端侧边栏改为顶部导航或悬浮菜单，腾出更多中央创作空间。1.2 隐藏复杂度 (化繁为简)配置项分层:主流程: 只保留“选品”、“语言”、“文案格式”三个核心输入。专家模式 (Advanced): 使用折叠面板隐藏“推理强度”、“联网增强”、“RAG 知识库”等技术参数。DIY 模式移除: 将“用户自填 Key 和 URL”的功能从主操作区移除，搬迁至“个人中心 -> 开发者配置”中。2. 商业化与计费模块2.1 额度系统 (Quota Management)展示组件: 在网页右上角（Header）增加个人额度仪表盘。显示内容: “剩余额度: 15 / 100 次”或“积分余额: 500 点”。收费方案支持:月费制 (Subscription): 针对长期企业用户，每月重置生成次数。计次制 (Top-up): 针对临时需求，支持购买次数加油包。限制逻辑:前端：额度不足时禁用生成按钮，并高亮“升级套餐”入口。后端：接入 LLM Lite 时同步校验用户 Session 对应的可用余额。2.2 统一模型抽象去供应商化: 屏蔽“字节、通义、GPT”等具体名称，对外统称为“企业级增强引擎”。智能调度: 基于你之前的 Testing Model 逻辑，后端/前端自动选择当前响应最快、可用性最高的供应商，用户只需点击“生成”即可。3. 全界面国际化 (i18n)3.1 架构实现框架: 使用 react-i18next 替换硬编码字符串。词典: 建立 src/locales/ 包含 zh-CN.json (中文) 和 en-US.json (英文)。自动检测: 网页版应支持根据浏览器语言自动切换首选语言。3.2 业务联动逻辑同步切换: 当用户切换 UI 语言为“English”时，系统应默认将“SEO 目标语言”也联动为“English”，减少用户二次选择。4. Web 版特有架构优化 (LLM Lite 接入准备)API 统一网关:Phase 1: 在前端维护供应商列表。Phase 2: 后端实现统一接入点。前端请求 api/v1/generate，携带用户 Token。Phase 3 (LLM Lite): 彻底解耦。前端只需发送 prompt，由 LLM Lite 完成模型路由、成本控制及错误重试。部署与安全:网页版需加入用户登录态校验（JWT 或 Session）。API Key 必须存放在后端，严禁在网页前端暴露供应商的原始 API Key。5. 任务清单 (Action Items for Antigravity)5.1 UI/UX 改造[ ] 切换全站为 SaaS White 主题，优化阴影与圆角。[ ] 实现 Step 3 的折叠面板，隐藏高级 AI 参数。[ ] 重构 Header，加入额度显示、语言切换、个人头像/设置。5.2 逻辑重构[ ] 接入 i18next 并将页面文字全部抽离到翻译文件。[ ] 实现额度校验逻辑：生成前判断次数，生成后更新余额。[ ] 优化 services/ai.ts，增加企业版统一请求封装（屏蔽多供应商选择）。5.3 网页版部署准备[ ] 针对移动端浏览器（手机端）进行响应式布局测试。[ ] 建立用户系统占位逻辑，为接入真实的 LLM Lite 后端做准备。
